Fortune Saver - Case Study

Responsive Web App

Description: A money saving tool that helps people save money quickly in preparation for a big purchase or expenditure.
Focus: user flow diagrams; low, mid, and high-fidelity wireframes; user feedback; user test analysis; mockups

My Process

Objective

(What) A responsive website, web app, and/or mobile app allowing all data on income and expenses to be recorded easily, on the go, and from a variety of devices. The tool displays data on the user's finances (how much money they spend and on what), and tells them what they can do to cut costs and save money in a certain amount of time.

Context

(Why) Saving money can be really hard, especially when you don't have a long time to do so. By providing personalized information on how a user can save based on their actual finances, the tool itself can provide financial advise during these finite periods of saving.

Target Audience

(Who) This tool is for people who want to save money, quickly, for a particular reason. For example, a wedding, big trip, or a new car. The users are not new to technology but do not like working with their finances. So, they require that the information they receive via the tool is accessible, and clearly tells them what they can do to better save.

Problem

  • As a new user, I need to be able to create a profile, so that the financial advise I receive is personalized to me, my spending habits, and tailored to my goals.
  • As a user, I want the money-saving tool I use to be motivational, so that I can stay on track with my goals.
  • As a user, I need to be able to tell the tool what my savings goal is and how long I have to reach it, so that I can save accordingly.

Solution

  • Having a responsive web app will make it easier for users to access across varying platforms.
  • Users will have the ability to create a profile.
  • Users will have the ability to add multiple savings goals.
  • Each goal will have a tracker to mark the user's progress.
  • Users will receive badges when completing goals.
  • Used FigJam for the user flow.

User Feedback

Scenario: You are planning on saving money for an upcoming trip and are looking to add a new savings goal through the Fortune Saver app. How would you go about this?

Participant Name: Mike N.
Task:

  • "I like it. If I were to use it to make financial goals, it seems easy to create and set those up. But it makes sense and I think that's important. There were enough cues that didn't make it confusing."
  • "I think the only part that I didn't know how would work was how would I be adding money towards the goal? Like idk if there needs to be another slide where you tap on an existing goal and you can see more details/options regarding it."
  • "But using the app itself was smooth and user friendly. The steps were logical for me."

Participant Name: Sarah G.
Task:

  • "So right off the bat I like that you can set a goal and see the goals that you have already achieved."
  • "Pretty straightforward, easy to understand."

Challenges

  • Incorporating color to the high-fidelity screens.
  • Coming up with the original user flow.
  • Visualizing what would be on the screens.

Workaround

  • Referenced the Color Interface Tool from Material.io
  • Drew multiple concepts and narrowed these down to decide on user-flow.
  • Referenced other financing apps and banking apps for inspiration.

Final Showcase

The original concept differed from the final results in that:

  • The hamburger menu was taken out in both the mid and high-fidelity screens. Added the app logo in its place.
  • The bottom and top navigation icons used were changed from white to black.
  • After receiving feedback on the mid- fidelity screens, incorporated some additional screens.

Takeaways

  • Would look at adding further screens in the second iteration of the app.
  • Would look to incorporate animation in the future (specifically with the goals and achievements screens).
